# Internacionalización (i18n)

Este proyecto utiliza **ngx-translate** para soportar múltiples idiomas (inglés y español).

## Configuración

### Archivos de traducción

Los archivos de traducción se encuentran en `src/assets/i18n/`:
- `en.json` - Traducciones en inglés
- `es.json` - Traducciones en español

### Estructura de traducciones

```json
{
  "header": {
    "title": "..."
  },
  "navbar": {
    "profile": "...",
    "skills": "...",
    "experience": "...",
    "projects": "...",
    "resume": "..."
  },
  "banner": {
    "name": "...",
    "title": "...",
    "contactMe": "..."
  },
  "sections": {
    "profile": {
      "title": "...",
      "description": "..."
    },
    ...
  },
  "experience": {
    "ipsos_developer": {
      "title": "...",
      "subtitle": "...",
      "description": "...",
      "info": "..."
    },
    ...
  },
  "projects": {
    "dsg_app": {
      "name": "...",
      "description": "..."
    }
  }
}
```

## Uso

### En templates HTML

Usar el pipe `translate`:

```html
<h1>{{ 'banner.title' | translate }}</h1>
<p>{{ 'sections.profile.description' | translate }}</p>
```

### En componentes TypeScript

Inyectar `TranslateService`:

```typescript
import { TranslateService } from '@ngx-translate/core';

constructor(private translate: TranslateService) {}

// Obtener traducción
const text = this.translate.instant('navbar.profile');

// Cambiar idioma
this.translate.use('es');
```

### Servicio de idioma

El proyecto incluye un servicio `LanguageService` que:
- Inicializa el idioma del navegador o el guardado en localStorage
- Proporciona métodos para cambiar idioma
- Persiste la preferencia del usuario

```typescript
import { LanguageService } from './core/services/language.service';

constructor(private languageService: LanguageService) {}

// Obtener idioma actual
const currentLang = this.languageService.getCurrentLanguage();

// Cambiar idioma
this.languageService.setLanguage('es');

// Alternar entre idiomas
this.languageService.toggleLanguage();
```

## Agregar nuevas traducciones

1. Agregar la clave y valor en ambos archivos (`en.json` y `es.json`)
2. Usar la clave en el template con el pipe `translate` o en el componente con `TranslateService`

## Selector de idioma

El botón de cambio de idioma se encuentra en el header de la aplicación y permite alternar entre inglés (EN) y español (ES).
